## System Requirements

Before installing QualiVision, ensure your system meets these requirements:

<CardGroup cols={2}>
  <Card title="Python Version" icon="python">
    Python 3.8 or higher required
    
    Verify: `python --version`
  </Card>
  
  <Card title="GPU Support" icon="microchip">
    CUDA-capable GPU recommended
    
    - DOVER++: ~12GB VRAM
    - V-JEPA2: ~16GB VRAM
  </Card>
  
  <Card title="Storage" icon="hard-drive">
    ~10GB for models and dependencies
    
    Additional space for datasets
  </Card>
  
  <Card title="Operating System" icon="desktop">
    Linux, macOS, or Windows
    
    Linux recommended for best performance
  </Card>
</CardGroup>

<Warning>
  While CPU-only execution is supported, GPU acceleration is **strongly recommended** for practical evaluation and training. CPU inference can be 10-50x slower.
</Warning>

## Installation Methods

<Steps>
  <Step title="Clone the Repository">
    First, clone the QualiVision repository from GitHub:
    
    ```bash
    git clone https://github.com/RITIK-12/QualiVision.git
    cd QualiVision
    ```
    
    This will download the complete framework including:
    - Model implementations
    - Training and evaluation scripts
    - Configuration files
    - Example notebooks
  </Step>
  
  <Step title="Install Dependencies">
    <Tabs>
      <Tab title="Standard Installation">
        Install all dependencies using pip:
        
        ```bash
        pip install -r requirements.txt
        ```
        
        This is the recommended method for most users.
      </Tab>
      
      <Tab title="Virtual Environment">
        Use a virtual environment for isolated installation:
        
        ```bash
        # Create virtual environment
        python -m venv venv
        
        # Activate (Linux/macOS)
        source venv/bin/activate
        
        # Activate (Windows)
        venv\Scripts\activate
        
        # Install dependencies
        pip install -r requirements.txt
        ```
      </Tab>
      
      <Tab title="Conda Environment">
        Use Conda for environment management:
        
        ```bash
        # Create conda environment
        conda create -n qualivision python=3.8
        
        # Activate environment
        conda activate qualivision
        
        # Install PyTorch with CUDA (Linux/Windows)
        conda install pytorch torchvision torchaudio pytorch-cuda=11.8 -c pytorch -c nvidia
        
        # Install remaining dependencies
        pip install -r requirements.txt
        ```
      </Tab>
    </Tabs>
  </Step>
  
  <Step title="Verify Installation">
    Verify that QualiVision is installed correctly:
    
    ```bash
    python -c "import torch; print(f'PyTorch: {torch.__version__}')"
    python -c "import torch; print(f'CUDA Available: {torch.cuda.is_available()}')"
    python -c "import transformers; print(f'Transformers: {transformers.__version__}')"
    ```
    
    Expected output:
    ```
    PyTorch: 2.0.0+cu118
    CUDA Available: True
    Transformers: 4.30.0
    ```
  </Step>
</Steps>

## GPU Setup

<Tabs>
  <Tab title="CUDA (NVIDIA)">
    For NVIDIA GPUs, ensure CUDA is properly installed:
    
    <Steps>
      <Step title="Check CUDA Version">
        ```bash
        nvidia-smi
        ```
        
        This shows your GPU and CUDA driver version.
      </Step>
      
      <Step title="Install PyTorch with CUDA">
        Visit [PyTorch Get Started](https://pytorch.org/get-started/locally/) and select your CUDA version:
        
        ```bash
        # For CUDA 11.8
        pip3 install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
        
        # For CUDA 12.1
        pip3 install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
        ```
      </Step>
      
      <Step title="Verify CUDA">
        ```python
        import torch
        print(f"CUDA Available: {torch.cuda.is_available()}")
        print(f"CUDA Version: {torch.version.cuda}")
        print(f"Device Count: {torch.cuda.device_count()}")
        print(f"Current Device: {torch.cuda.current_device()}")
        print(f"Device Name: {torch.cuda.get_device_name(0)}")
        ```
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="ROCm (AMD)">
    For AMD GPUs with ROCm:
    
    ```bash
    # Install PyTorch for ROCm
    pip3 install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/rocm5.6
    ```
    
    Check compatibility at [PyTorch ROCm](https://pytorch.org/get-started/locally/).
  </Tab>
  
  <Tab title="MPS (Apple Silicon)">
    For Apple Silicon (M1/M2/M3) Macs:
    
    ```bash
    # Standard PyTorch installation includes MPS support
    pip3 install torch torchvision torchaudio
    ```
    
    Verify MPS:
    ```python
    import torch
    print(f"MPS Available: {torch.backends.mps.is_available()}")
    print(f"MPS Built: {torch.backends.mps.is_built()}")
    ```
    
    <Note>
      Use `--device mps` when running evaluation scripts on Apple Silicon.
    </Note>
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="CUDA Out of Memory" icon="memory">
    If you encounter OOM errors:
    
    1. Reduce batch size:
       ```bash
       python scripts/evaluate.py --model dover --batch-size 1 --data data/test
       ```
    
    2. Use gradient checkpointing (for training)
    
    3. Close other GPU applications
    
    4. Use CPU if necessary:
       ```bash
       python scripts/evaluate.py --model dover --device cpu --data data/test
       ```
  </Accordion>
  
  <Accordion title="Flash Attention Installation Issues" icon="bolt">
    If `flash-attn` fails to install:
    
    1. Skip flash attention (optional dependency):
       ```bash
       pip install -r requirements.txt --no-deps
       pip install <packages except flash-attn>
       ```
    
    2. Or install from source:
       ```bash
       pip install flash-attn --no-build-isolation
       ```
    
    3. Flash attention is optional for inference, mainly benefits training speed
  </Accordion>
  
  <Accordion title="Decord Video Loading Errors" icon="video">
    If video loading fails:
    
    1. Ensure FFmpeg is installed:
       ```bash
       # Ubuntu/Debian
       sudo apt-get install ffmpeg
       
       # macOS
       brew install ffmpeg
       
       # Windows
       # Download from https://ffmpeg.org/download.html
       ```
    
    2. Test decord:
       ```python
       import decord
       vr = decord.VideoReader('path/to/video.mp4')
       print(f"Frames: {len(vr)}")
       ```
  </Accordion>
  
  <Accordion title="Transformers Model Download Issues" icon="download">
    If model downloads are slow or fail:
    
    1. Use Hugging Face mirror (China):
       ```bash
       export HF_ENDPOINT=https://hf-mirror.com
       ```
    
    2. Pre-download models:
       ```python
       from transformers import AutoModel
       AutoModel.from_pretrained("BAAI/bge-large-en-v1.5")
       ```
    
    3. Set cache directory:
       ```bash
       export HF_HOME=/path/to/cache
       ```
  </Accordion>
</AccordionGroup>

## Verify Installation

Run this comprehensive verification script:

```python
import sys
import torch
import transformers
import torchvision
import decord
import cv2
import numpy as np
import pandas as pd

print("QualiVision Installation Check")
print("=" * 50)

# Python version
print(f"Python: {sys.version}")

# PyTorch
print(f"\nPyTorch: {torch.__version__}")
print(f"CUDA Available: {torch.cuda.is_available()}")
if torch.cuda.is_available():
    print(f"CUDA Version: {torch.version.cuda}")
    print(f"GPU: {torch.cuda.get_device_name(0)}")
    print(f"GPU Memory: {torch.cuda.get_device_properties(0).total_memory / 1e9:.2f} GB")

# Transformers
print(f"\nTransformers: {transformers.__version__}")

# Other key libraries
print(f"\nTorchvision: {torchvision.__version__}")
print(f"OpenCV: {cv2.__version__}")
print(f"NumPy: {np.__version__}")
print(f"Pandas: {pd.__version__}")
print(f"Decord: {decord.__version__}")

print("\n✓ All dependencies installed successfully!")
```

Save as `check_install.py` and run:
```bash
python check_install.py
```
